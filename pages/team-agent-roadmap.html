GOAL
Agent 研发团队全员都要往独当一面的 Agent 研发方向发展 —— 能独立设计、独立交付、独立沉淀，而非只会跟着无代码引擎拖拽。
01 内部营造氛围 + 挖掘筛选 In-house
团队内营造 Agent 研发转变氛围，让每个人从"用工具"转向"造 Agent"。
落地方案
每个人开发自己的一个 Agent 分身 —— 把自己日常重复性工作交给分身，倒逼 Agent 研发能力生长。
机制 · Review
每周 Review 每个人的进展，跟踪分身能力成长曲线，以便后续发掘与筛选谁具备独当一面的潜质。
立项→ 开发→ 每周 Review→ 筛选
三赢
WIN · 1 研发提效 分身接管重复劳动
WIN · 2 学习提升 边造边学 Agent 研发
WIN · 3 工作 AI Native 研发流程原生 AI 化
02 校招补位 · 备选培养 Campus
内部发掘可能 达不到公司发展预期，需引入 1-2 年经验的 Agent 初级研发 作为培养备选。
为什么是应届 1-2 年
没有很多历史思想包袱 —— 未被 SaaS 时代的"确定性 API + 状态机"思维固化，更容易长出 概率输出 + 反思循环 的 Agent 直觉。
筛选标准
  • 985 科班：计算机/AI 相关专业，基础扎实
  • 1-2 年经验：有工程能力，但不深陷旧范式
  • 无历史 SaaS 包袱：未被传统软件交付模式定型
  • 对 Agent 有热情：自驱动学习，不止是来打工
定位 · 培养备选
作为独当一面 Agent 研发的培养备选池，与策略 1 的内部筛选形成双轨：能从内部长出来的优先，长不出来的位置由校招补位填上。
