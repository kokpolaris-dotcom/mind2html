# Wanpix.ai 战略与技术架构

> 文旅营销域 · 能力基建 — 营销域专家能力的数字化分身

---

## 第一部分:业务定位

### 核心定位

**Wanpix.ai 不是降本工具,而是文旅行业营销域专家能力的数字化分身。**

它解决的不是生产效率问题,而是营销能力缺口——将深域专家经验数字化为可规模化复制的 Agent 能力。

Wanpix.ai 不是纯粹的"降本工具"。它的核心,是把专业代运营团队的营销域核心能力产品化、Agent 化——解决文旅行业最痛的"营销能力断层",实现专家级能力的平权与规模化复制。

---

### 一、文旅行业的三层力量

从"谁能解决营销问题"的视角,文旅行业存在三类角色,各自卡在不同环节。

#### 1. 上游 & 中游 — 资源方 & 服务商

- **标签**:有产品,但无营销能力
- **核心痛点**:景区、酒店、体验服务商多是"资源型 / 服务密集型"企业。擅长做服务、搞基建,面对抖音、小红书等新媒体极度缺乏专业营销域能力——不懂抓热点、不懂写爆款脚本、不懂流量转化。
- **业务解法**:通过代运营服务或数字化产品弥补内容营销、流量获取与转化闭环的短板,把"资源"转化为"客流"。

#### 2. 传统代运营团队 — 人力驱动

- **标签**:解决 0→1 冷启动,但难规模化
- **核心痛点**:人力代运营是非标、重资产、难规模化。金牌运营一年能服务的客户有限,人员流动导致服务质量波动,边际成本极高。
- **业务解法**:提供定制化专业策略,但难以覆盖全行业海量商家的普惠需求。

#### 3. Wanpix.ai — 营销域能力 Agent 化

- **标签**:专家级能力的规模化复制
- **核心痛点**:行业需要"懂文旅营销的专家",但专家太贵且太少。纯工具不懂业务,纯人力跑不快。
- **业务解法**:把"金牌运营"的人脑经验,转化为 AI Agent 的执行逻辑——永不离职、24h 在线、拥有全行业顶尖营销经验。将营销专家的深度认知和操作逻辑封装为 Agent,实现标准化的高质量交付,以数字化的方式跨越人力瓶颈。

### 二、三大核心价值点

Wanpix.ai 把"人"的核心价值(策略、洞察、创意逻辑)抽离出来,用 Agent 规模化执行——这是一条从"经验"到"能力基建"的闭环。

#### ① 知识沉淀 — 营销域知识的结构化

把代运营团队实战中积累的**隐性 Know-how**——什么样的景区文案能爆、什么时间发视频转化率高、如何设计团购套餐钩子——提炼成标准化的策略模型与知识图谱,成为 Agent 决策的底层认知。

#### ② Agent 封装 — 核心能力的 Agent 化

把"人脑经验"转化为"Agent 执行逻辑"。它不再只是生成文字,而是具备**营销思维**——像资深运营一样自动策划营销动作、生成平台调性内容并执行分发。从"知道怎么做"到"系统自动做"。

#### ③ 标准化交付 — 交付的标准化与规模化

把依赖"金牌运营个人能力"的非标服务,变成可无限复制的**标准化数字服务**。商家等于以极低成本雇佣了一支顶尖的超级数字运营团队。让每一次交付都保持一致的高水准,同时突破产能天花板。

### 四、双层级产品架构

把"能力"变成可组装的"乐高积木"——商业天花板从 SaaS 工具跃升到 **PaaS 生态平台**。

#### Layer 1 — 底座平台(能力原子化 · 行业基建)

文旅营销领域的"安卓系统",提供最底层的行业水电煤。Harness 技术(RAG / Skill / Tools)本质是把代运营团队积累的"营销域核心能力"进行结构化拆解与原子化封装——文旅营销领域的"操作系统"。

- **RAG 行业知识库**:沉淀爆款文案库、平台算法规则、历史转化数据等"隐性知识"。
- **Skill 标准化技能**:封装具体营销动作——热点捕捉、脚本生成、团购套餐设计。
- **Tools 执行接口**:打通抖音、小红书等平台 API,解决"最后一公里"分发。

#### Layer 2 — 官方专家分身(标准化交付 · 标杆示范)

利用底座 Harness 能力,组装出"开箱即用"的标准化产品。解决行业 80% 的通用需求——是底座的**旗舰应用**与样板间。

- 面向**中小商家**——无需懂底层逻辑,直接带来客流的"超级数字员工"。
- 既是**直接营收来源**,又向市场证明底层能力的强大。

#### Layer 3 — 用户自定义分身(生态化 · 长尾赋能)

允许大型文旅集团、MCN、资深代运营调用底座 Harness 能力,注入**私有认知**(品牌调性、私域逻辑、客群洞察),构建专属 Agent。

- **长尾个性化需求**:文旅是非标行业,头部大客户有极个性化营销逻辑,官方分身无法完全覆盖。
- **Lock-in 粘性效应**:大客户用底座 RAG 与 Tools 搭建专属工作流后,迁移成本极高。
- **反哺底座飞轮**:自定义产生的新玩法、新认知,可被提炼回底座,形成生态飞轮。

### 五、从"造车"到开放"造车平台"

双层级架构让 Wanpix.ai 在不同客户眼中扮演不同角色——这是极具想象力和壁垒的 ToB 模式。

> 不仅自己造"车"(官方分身),还开放了顶级的"造车平台"(Harness 底座)。这在 ToB 业务中是一个极具想象力和壁垒的商业模式。

#### 对中小商家 — 营销专家分身提供商

卖**标准 SaaS / 服务**。以极低成本雇佣一支"永不离职、24h 在线、拥有全行业顶尖营销经验"的超级数字运营团队。

标签:SaaS · 标准化交付 · 旗舰应用

#### 对大型文旅集团 / MCN — 文旅营销能力的 PaaS 平台

卖**底层能力与生态**。不仅自己造"车"(官方分身),还开放顶级"造车平台",允许别人造出符合自己需求的"特种车辆"。

标签:PaaS · Harness 开放 · 生态飞轮 · 高 Lock-in

## 第二部分:技术架构

业务域已清晰 → 技术域如何落地。

围绕双层级产品架构(底座 + 官方 / 自定义分身),技术上由**五大核心模块**配套:编排引擎、知识中台、技能工具层、Agent Builder、数据飞轮——形成"感知 → 决策 → 执行 → 反馈 → 进化"的完整闭环。

把营销域的**能力**、**知识**、**执行**,沉淀为可编排的**底座**。

---

### 〇、架构总览 — 五大模块如何咬合

- **横向**是业务纵深:应用层(分身)→ 构建层(Builder)→ 编排层(大脑)→ 能力层(RAG + Skill/Tools)
- **纵向**是基础设施柱:多租户、权限、计费、数据飞轮贯穿全部四层。

Agent Builder 位于分身与大脑之间——它是"造车平台",在分身(车)和编排引擎(引擎)之间做组装。

#### 横向业务纵深(四层)

**L4 · 应用层 — Agent 分身**(最终交付物,商家直接使用的"超级数字员工")
- 用户自定义分身:注入私有认知(品牌调性 / 私域逻辑 / 客群洞察),调用"行业库 + 私有库"。
- 官方专家分身:基于 Harness 组装,开箱即用,调用官方行业库与官方 Skill 集。

**L3 · 构建层 — Agent Builder**(· 造车平台,用户 DIY 专属分身的前端入口)
- Visual Editor:拖拽式工作流,把 RAG / Skill / Tools 像搭积木一样组合。
- Prompt IDE:Few-shot 模板,上传品牌优秀文案让 Agent 学会调性。
- Simulation:发布前用历史数据跑沙盒,评估质量与合规性。

**L2 · 编排层 — Orchestration Engine**(· 大脑,把 Harness 能力组装为可执行工作流)
- Multi-Agent:基于 Mastra 的多 Agent 协作,DAG 工作流,选题 / 文案 / 分发流水线。
- Planner:自然语言指令 → 任务自动拆解,调用底层 Skills 与 Tools。
- State Mgmt:长链路任务的上下文记忆,多轮对话中不丢信息。

**L1 · 能力层 — RAG + Skill/Tools**
- RAG 2.0 知识中台(· 记忆库):区分官方知识与用户私有认知。
  - 混合检索:向量 + BM25 + 知识图谱
  - 动态更新管道:ETL 实时抓取热点 / 规则
- Skill / Tools 层(· 手脚):原子能力封装 + 平台对接
  - 标准 Skill 协议:JSON Schema / OpenAPI
  - API 网关 + 沙箱:限流 / 鉴权 / 隔离执行

#### 纵向基础设施柱(Cross-Cutting · 贯穿 L1–L4)

- **Multi-Tenancy 多租户隔离**:数据 / 知识 / Skill 全链路租户隔离,官方库与私有库物理分隔。
- **IAM / RBAC 权限管理**:角色 / 资源 / 动作的细粒度授权,分身、Skill、数据的访问控制。
- **Billing 计费体系**:按 Agent 调用 / Token / Skill 执行计量,支持套餐与按量计费。
- **Data Flywheel 数据飞轮**:全链路埋点 + RLHF + A/B Testing,把执行结果反哺为新的能力。

#### 典型调用流 — 一次请求如何穿过各层

以商家输入"帮我策划一个五一假期的景区促销"为例:

1. **① 接收 · L4** — 分身接收自然语言指令,传递给编排引擎
2. **② 规划 · L2** — Planner 拆解任务,编排多 Agent 工作流
3. **③ 检索 · L1** — RAG 混合检索五一热点 / 景区知识 / 历史 SOP
4. **④ 执行 · L1** — Skill 生成脚本,Tools 分发到抖音 / 小红书
5. **⑤ 回流 · 贯穿** — 飞轮记录结果,A/B 验证后沉淀为官方 Skill

> 贯穿全程:多租户保障数据隔离、权限校验每次调用、计费计量各层消耗、飞轮回流结果。

### 一、Agent Orchestration Engine — 编排引擎

底座的"大脑"。解决"如何把 Harness 能力组装起来"的问题。

#### ① Multi-Agent Framework — 多 Agent 协作框架

基于 **Mastra**(TypeScript 原生 Agent 框架)搭建多智能体协作体系。支持 **DAG 有向无环图**工作流编排,让"选题 Agent""文案 Agent""分发 Agent"像流水线一样协同工作;并通过其原生的 Agent Network 与 Workflow 能力,支持顺序、并行、循环等多种协同模式。

#### ② Dynamic Planner — 动态任务规划

底层需要一个强大的 Planner,能够根据用户的自然语言指令——比如"帮我策划一个五一假期的景区促销"——自动拆解任务,并调用底层的 Skills 和 Tools。

#### ③ State Management — 状态管理

文旅营销是**长链路任务**,引擎必须能精准记忆上下文状态,确保在多轮对话和复杂任务执行中不丢失信息。

---

### 二、RAG 2.0 与向量知识中台

底座的"记忆库"。**区分官方知识与用户私有认知**的关键。

#### ① Multi-Tenant Isolation — 多租户知识隔离

在向量数据库(**Milvus** / **Qdrant**)层面实现严格的 **Namespace 命名空间**隔离。官方分身调用行业通用库,自定义分身调用"行业库 + 自有私有库"。

#### ② Hybrid Retrieval — 混合检索策略

文旅行业有大量专有名词(特定景区 IP、非遗名称),纯向量检索易出错。采用 **向量检索 + 关键词检索(BM25)+ 知识图谱** 的混合模式,提高召回准确率。

#### ③ Live Update Pipeline — 知识动态更新管道

文旅热点变化极快。需搭建**自动化 ETL 管道**,实时抓取全网文旅热点、平台算法规则变化,自动清洗并向量化注入知识库。

---

### 三、Skill / Tools 注册与网关

底座的"手脚"。实现**能力的原子化封装**。

#### ① Standard Protocol — 标准化 Skill 协议

定义一套标准的 Skill 描述协议(基于 **JSON Schema** 或 **OpenAPI**)。无论官方"脚本生成 Skill",还是用户上传的"私域客服 Skill",都必须遵循该协议才能被 Agent 调用。

#### ② API Gateway — API 聚合网关

统一管理对抖音、小红书、微信等第三方平台 API 的调用。做好**限流、鉴权、异常重试**机制,避免因某个平台接口波动导致整个 Agent 瘫痪。

#### ③ Sandbox Execution — 沙箱执行环境

当用户自定义分身调用复杂代码或工具时,必须在**隔离的沙箱环境**中执行,防止恶意代码或死循环拖垮整个底座平台。

---

### 四、低代码 / 无代码 Agent Builder

实现**"用户 DIY 专属分身"**的前端交互核心。

#### ① Visual Workflow Editor — 可视化工作流编辑器

提供类似 **Coze** / **Dify** 的拖拽式界面。用户可通过连线方式,把底座的 RAG、Skill、Tools 像**搭积木**一样组合起来。

#### ② Prompt IDE — 提示词工程辅助

可视化的 Prompt 调试界面,帮助用户注入自己的"认知"。提供 **Few-shot 少样本学习**模板——用户上传自己品牌的优秀文案作为范例,Agent 就能快速学会这种调性。

#### ③ Simulation Sandbox — 仿真测试环境

在用户发布自定义 Agent 前,提供**仿真沙盒**,用历史数据跑一遍,评估其生成内容的质量和合规性。

---

### 五、Data Flywheel 数据飞轮

确保**"官方分身越来越聪明,底座越来越强"**的闭环。

#### ① Full-Chain Tracing — 全链路埋点与追踪

记录 Agent 的每一次**思考链(CoT)**、每一次工具调用、以及最终的业务结果(视频播放量、核销率)。

#### ② RLHF Interface — RLHF 人类反馈接口

在官方分身和自定义分身后台,提供便捷的**点赞 / 点踩 / 修改**接口。用户反馈自动回流,用于微调底层 LLM 或优化 RAG 检索权重。

#### ③ A/B Testing — A/B Testing 框架

支持对不同 Agent 策略进行灰度测试,自动筛选出**转化率最高的营销 SOP**,并将其固化为新的官方 Skill。

#### 飞轮闭环(Flywheel Closed Loop)

1. **STEP 1 · Agent 执行** — 编排 + 知识 + 工具
2. **STEP 2 · 行为埋点** — CoT + 调用日志
3. **STEP 3 · 业务结果** — 播放量 / 核销率
4. **STEP 4 · 反馈回流** — RLHF + A/B
5. **STEP 5 · 沉淀为 Skill** — 反哺底座 · 进化

> 最优 SOP 经 A/B 验证后固化为新的官方 Skill,回到 STEP 1 形成闭环。

### 六、技术与业务架构的映射

每个技术模块都对应业务架构中的具体能力。

| 技术模块 | 角色定位 | 支撑业务能力 | 对应业务层 |
|---|---|---|---|
| ① Orchestration Engine(编排引擎) | 大脑 | 将 Harness 能力组装为可执行工作流 | 底座平台核心 |
| ② RAG 2.0 知识中台(向量知识中台) | 记忆库 | 官方行业库 + 用户私有库的隔离与检索 | 底座 · 自定义分身 |
| ③ Skill / Tools 层(技能工具层) | 手脚 | 原子化营销动作 + 平台 API 分发 | 底座 · 官方 / 自定义分身 |
| ④ Agent Builder(低代码搭建器) | 交互层 | 用户 DIY 专属分身 + 注入私有认知 | 用户自定义分身 |
| ⑤ Data Flywheel(数据飞轮) | 进化引擎 | 埋点 + RLHF + A/B 反哺 Skill 沉淀 | 横切 · 全层受益 |

#### 官方专家分身的技术依赖

- ① 编排引擎 + ② 官方行业库 + ③ 官方 Skill 集
- ⑤ 飞轮回流让官方 Skill 持续升级
- 不依赖 ④ Agent Builder——官方分身是底座自带的旗舰应用

#### 用户自定义分身的技术依赖

- ④ Agent Builder + ② 私有 Namespace + ③ 用户上传 Skill
- ① 编排引擎调度混合调用(官方 + 私有)
- ③ 沙箱执行 / ④ 仿真测试保障安全合规
